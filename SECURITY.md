# Security Policy

## Reporting a Vulnerability

Do not open a public issue for undisclosed vulnerabilities.

1. Send a private report to project maintainers.
2. Include affected versions, reproduction details, and impact.
3. Include any workaround or mitigation if known.

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage and severity assignment: within 7 business days
- Remediation plan or workaround: as soon as practical based on severity

## Scope

This policy applies to:

- `supply-chain-framework`
- `script-audit`
- Published OMSF rule packs and policies

## Coordinated Disclosure

Please allow maintainers a reasonable remediation window before public disclosure.
